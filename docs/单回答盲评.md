# 单回答盲评 · 功能文档

web 评估台（`python -m auto_eval.web.server` → http://localhost:8501）四个模式之一。对**用户提供的单个回答**做盲评打分；若额外提供竞品答案，产品专家裁判会以其为对比参考。

## 一、功能概述

- **盲评**：裁判全程看不到参考答案，独立判断回答好坏。
- **多裁判集成**：可勾选多个裁判（研发人员 / 终端用户 / 产品专家），各自打分后聚合（多数投票 + 一致率检测）；分歧大时主席仲裁。
- **裁判 agent loop**：裁判自主联网查证（`web_search` 多源聚合 / `fetch_page` / `calculate` / `python_run`），核查事实/计算，不只凭印象。
- **维度打分理由**：每个一级维度除分数外，还输出**打分理由**（结构化 `rubric_reasons`），拼进"理由"字段、前端 hover 可见。
- **按垂域分析**：每题归一到垂域（Skill），跑完给分垂域总览 + 维度问题分布柱状图 + 样本分布饼图 + 题号下钻。
- **元评测（可选）**：填了参考答案的题，用客观真值（EM/F1/语义相似度）校验裁判判得准不准。
- **实时 + 多格式导出**：SSE 逐题推送；导出 CSV / JSON / XLSX（按垂域分 sheet）。
- **历史记录**：本地持久化，可加载历史任务、可删除。

## 二、输入

### 输入方式
粘贴文本 / 上传文件（`.jsonl` / `.txt` / `.csv` / `.json`）。

### 文本格式（每行一题，`|||` 分隔）
```
query ||| answer [||| competitor] [||| reference]
```
- `competitor`：竞品答案，可选；填了**产品专家**裁判会作对比参考（其他裁判忽略它）。
- `reference`：参考答案，可选；填了跑元评测。

示例：
```
光合作用是什么？ ||| 植物利用光合成有机物 ||| 竞品：植物吸收光能合成有机物放出氧气 ||| 绿色植物利用光能将二氧化碳和水合成有机物并释放氧气
中国最长的河流？ ||| 长江
```

### JSONL 格式（每行一个 JSON 对象）
| 字段 | 必填 | 说明 |
|---|---|---|
| `question`（或 `query`） | ✅ | 题目 |
| `answer` | ✅ | 待评回答 |
| `competitor` | ⬜ | 竞品答案（产品专家用） |
| `reference` | ⬜ | 参考答案（元评测用） |
| `category` | ⬜ | 垂域类目（如"汽车"/"计算"）；不填则裁判自动分类 |

> 缺 `answer` 的行报错跳过。解析后显示"预览（N 题）"。

## 三、评估配置

| 配置 | 说明 |
|---|---|
| **裁判** | 多选，中文显示名：**研发人员**（strict_expert）/ **终端用户**（end_user）/ **产品专家**（product_expert）；至少选一个 |
| **产品专家** | 勾选后，**未提供 competitor 的题会被跳过**（产品专家靠对比参考打分） |
| **并发** | 1–16，裁判 agent loop 含联网查证，过高易触发限流 |

## 四、输出

### 1. 实时逐题结果（SSE）
每题评完即推送到结果表，无需等全部完成；进度条 `已完成/总数`。

### 2. 结果表格（每题一行）
题号 / 垂域 / 题目 / 回答 / **判定** / **总分** / 各一级维度分 / 联网 / 截断 / 仲裁 / 与真值 / **理由**。

- **判定**：`right` / `wrong` / `partial` / `unclear`（无法确定时不硬猜）。
- **各维度分**：每个一级维度一列；**hover 单元格显示该维度的打分理由**（`rubric_reasons`）。
- **理由**列：一句话总判定 + 各维度理由拼接（`总理由 || 维度A：理由A ｜ 维度B：理由B`）。
- 部分题下方展开 **🔍 裁判查证轨迹**（web_search/fetch_page 调用链）。

### 3. 总结区（全部完成后）
完成 `done/total` / 平均总分（归一化%）/ 裁判判题准确率（有参考答案题）。若有题落"通用"兜底，顶部显示**全局分类失败提示**。

### 4. 📊 按垂域分析（全部完成后）
- **饼图**：垂域样本量分布。
- **总览表**：每垂域一行（样本量 / 正确率 / 平均分 / 首要错因 / 低一致率 / 分类失败占比）。
- **维度问题分布柱状图**：每个有问题的垂域一张，x=一级维度、y=问题占比（维度分 ≤ 阈值算"问题"，默认 ≤2，可配 `ensemble.dim_problem_threshold`）。
- **题号下钻**：每个维度的问题占比关联题号清单，点击可筛到具体题核查。

### 5. 导出
- **CSV**（UTF-8-SIG）：逐题扁平（维度分塞单列）。
- **JSON**：含 `results`（逐题，含 `rubric_reasons`）/ `summary`（含 `by_skill` 垂域分析）。
- **XLSX**：多 sheet —— 运行信息 / 逐题结果(概览) / **逐题-{垂域}**（每垂域一个 sheet，同垂域维度一致、分+理由成对列）/ 评估失败 / 垂域总览 / 维度问题占比 / 图表数据 / 汇总指标。纯数据（不含图表，图表看 web 端 ECharts）。

## 五、评估原理

```
输入(query + answer [+competitor] [+reference])
  → 解析为题目列表
  → 逐题盲评：裁判 agent loop
      · 多源聚合 web_search（tavily/jina 等并行 + 交错 + 去重 + 来源标注）
      · 维度打分（一级编号、二级缩进）+ 每维度打分理由
  → 多裁判集成（多数投票）→ 分歧大则主席仲裁
  → [有 reference] 元评测：盲判 vs 客观真值 → 判题准确率
  → 垂域归一（SkillRouter）→ 按垂域聚合 + 维度问题分布 + 错因聚类
  → 实时输出逐题结果 + 总结 + 垂域分析 + 导出
```

**关键原则**：
- **盲评纯洁性 / 两类核查区分**：①**断言核查**（"X 是 Y 的 Z"、数字/日期/参数）→ 带断言验证真伪；②**开放身份题**（"X 是谁/是什么"）→ 必须**先用纯实体名搜索**（不带答案里的机构/头衔），看有哪些可能、是否有同名，再对照答案——严禁把答案归属带进 query，否则自我证实 + 漏同名。
- **判定诚实**：无法确定对错时给 `unclear`（不硬猜）；`partial` 仅用于"方向对但不完整/有小错"。
- **垂域归一**：每题经 `SkillRouter.resolve` 归一到垂域（名用 `display` 中文）；未标 category 时裁判自动分类（带"通用"出口，分类失败落 default）。来源标记 `category_source`：`dataset`（数据集自带）/ `auto_classified`（裁判分类）/ `fallback_default`（落通用）。
- **多源聚合搜索**：`search_provider` 支持单值或列表（`[tavily, jina]`），并行调所有源、交错合并、按 url 去重、来源标注；缺 key 的源自动跳过。

## 六、历史记录

- 评测结果持久化在 `runs/web_history/{task_id}.json`，服务重启可加载。
- 历史记录表可「加载」（还原结果）、「删除」（物理删 json，二次确认；删当前任务同时清空结果区）。
